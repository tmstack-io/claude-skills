# claude-skills

Claude Code 専用スキル群（および `~/.claude` 配下の codex 起用ルール）が共有する運用語彙のグロッサリ。実装詳細は各 SKILL.md が正本であり、ここには書かない。

## Language

### codex 実行

**事後判定**:
codex のタイムアウト到達後に、進捗ログの最終更新からの経過時間で原因をハングか処理時間不足かに二値分類すること。
_Avoid_: タイムアウト解析、原因調査

**ハング**:
codex が無活動のまま時間切れになった状態。同一条件の再実行で回復することが多い。
_Avoid_: フリーズ、スタック

**処理時間不足**:
codex が直前まで進捗を出力し続けたままタイムアウトに達した状態。正当な長時間処理の打ち切りであり、再実行にはタイムアウトの延長が要る。
_Avoid_: 時間切れ（ハングと区別しない用法）

**Claude 代替**:
codex が利用不能または失敗した工程で、Claude 自身が同一観点の評価を代行すること。報告には代替である旨を必ず明記する。
_Avoid_: フォールバック評価、自前レビュー

### 配役と輸送路

**配役**:
セッション中、特定ロールのタスクを外部エージェントへ割り当てるセッションモードの働き（現行実体: concertino-codex と、その編成1ラッパー solista-codex）。
_Avoid_: キャスティング、ルーティング

**編成**:
セッションに立てる奏者（配役先 codex インスタンス）のロールと人数の宣言。奏者合計の上限は4。
_Avoid_: 構成、座組

**指揮者**:
編成へタスクを分割・配分し、質問に答え、成果を検収する Claude 本体の働き。maestro はこの働きをセッション構造として固定する別スキル。
_Avoid_: オーケストレーター、コーディネーター

**パート譜**:
implement 奏者一人に排他的に割り当てられた担当ファイル集合。パート譜の外への書き込みは停止→質問→指揮者の再配分で扱う。
_Avoid_: 担当範囲、スコープ

**ロール**:
配役の単位。implement（書き込みを伴う実装）・review（批評）・explore（読み取り調査）の3種。
_Avoid_: 役割分担、モード

**輸送路**:
配役先エージェントとの通信経路。観戦・介入可能な **TUI 輸送路**（herdr ペイン）と、headless の **CLI 輸送路**（結果をファイルで受ける非対話実行）の2種。
_Avoid_: トランスポート、チャネル

**single-writer**:
どのファイルも、それへ書ける主体を常に一つに限る規律（編成1では「作業ツリーへ書けるのは一人」という従来形に一致する）。
_Avoid_: 排他制御、ロック

### レビューループ

**GREEN**:
レビューループの合格判定。指定 severity 以上の指摘がゼロである状態。
_Avoid_: 合格、クリア、pass

**レビュー契約**:
レビュー1回分の依頼（対象・観点・却下履歴）と応答（severity 付き指摘列挙または GREEN）の固定形式。正本は iterate-review。
_Avoid_: レビューインターフェース、レビュー形式

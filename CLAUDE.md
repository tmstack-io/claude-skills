# claude-skills — Claude Code 専用スキル正本リポジトリ

自作 Claude Code **専用**スキルの正本。`~/.claude/skills/` の同名エントリはここへのシンボリックリンクであり、どちらのパスを編集しても同じ実体に反映される。ここに置くスキルは、サブエージェントの並列実行・独立コンテキストの多角性・codex との相互チェックなど、Claude Code での運用構造そのものが品質の源泉であり、意図的に他エージェント向けの汎用化をしない（汎用化するスキルは agent-skills リポジトリに置く）。新しいマシンへの展開手順と収録一覧は README.md を参照。

## 編集時の前提

- 新規スキルは本リポジトリ直下に `<スキル名>/SKILL.md` として作成し、`ln -s` で `~/.claude/skills/` にリンクを張る。
- スキルを追加・改名・削除したら README.md の収録スキル表を更新する。
- スキルの査読・洗練には、Claude Code 上で agent-skills の `/skill-refine` を **`--myself`** を付けて使う（対象を実行エージェント＝Claude Code の専用スキルとして査読する。汎用スキル基準は適用しない）。
- 設計セッションで確定した横断用語（複数スキル・複数セッションで使う語彙）はリポジトリ直下の `CONTEXT.md`（グロッサリ。実装詳細は書かない）に記録する。無ければ最初の用語確定時に作成する。覆しにくく・理由が自明でなく・実際のトレードオフの結果である設計判断は `docs/adr/` に ADR として残す（3条件が揃わなければ作らない）。

## スキル設計規約（本リポジトリの全スキル共通）

- **自己完結**: 依存してよいのは Claude Code 本体機能・標準ツール・外部 CLI（codex 等、存在チェックとフォールバック付き）のみ。プラグインへの参照は書かない。Claude Code 固有のツール名（Agent / AskUserQuestion 等）はそのまま使ってよい。
  - 例外: **agent-skills リポジトリのスキルへの参照は可**（現行例: deep-pr-review → clarify-ja）。参照は「対象 SKILL.md を Read し、このコンテキスト内で規則を適用する」形とし、Skill ツール経由の別呼び出しにはしない。リポジトリを跨ぐ依存のため、解決順（自スキルの隣 `../<名前>/SKILL.md` → Claude Code のスキルディレクトリのプロジェクト側 → グローバル側、すなわち `.claude/skills/` → `~/.claude/skills/`）と、見つからない場合の**前置き中止ゲート＋復旧手順**を SKILL.md に必ず明記する。
- **起動方式の選択**: 自然文の依頼で発火させたい・他スキルから到達される → モデル起動型（description に「何をするか＋トリガー1分岐1つ」）。手動専用・起動＝実行許可となる重い操作 → `disable-model-invocation: true`（description はヒト向け1行要約、トリガー羅列なし）。
- **出力の安定性**: 各ステップに done / not-done を判定できる完了条件を付ける。曖昧な判断ゲートは二値化する（例: codex-review-loop の GREEN）。網羅が重要な箇所は「全項目が〜」の形で漏れゼロを要求する。
- **安全ゲート**: 破壊的・不可逆な操作は dry-run 既定＋承認ゲート。フォールバックした場合は代替であることを報告に明記する（実施済みと詐称しない）。
- **git への態度を明示する**: ループ・自動修正系のスキルは、git 履歴に触れるか否かを必ず規定する（明示しないと実行側がコミット等の履歴操作を発明する）。あわせて非 git 環境での挙動（対象確認・バックアップ等）も定義する。現行例: codex-review-loop の「git 規律」。
- **単一正本**: 同一の意味を複数箇所に書かない。再掲が必要なら正本を1箇所決めて参照する。サブエージェントへ転記する判定基準は `BEGIN/END` マーカーで区切り「一字一句そのままコピー」と指示する。
- **肯定形で書く**: 目標挙動を述べ、禁止形は肯定形で表せないハードガードレールに限る（その場合も代替行動を併記）。
- **環境・プロジェクト非依存**: 特定マシンの絶対パス・特定ユーザー名・特定プロジェクトの固有知識を書かない（Claude Code の標準パス `~/.claude/` 等への依存は本リポジトリでは可）。プロジェクト固有情報は「実行時に CLAUDE.md / rules / package.json 等から収集する手順」として書く。
